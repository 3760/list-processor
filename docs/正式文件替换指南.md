# 正式文件替换指南

## 概述

本指南说明如何将开发/测试阶段的 mock 文件替换为正式文件，以便进行生产环境部署。

## 文件替换清单

### 1. 字段规范文件（必须）

**源文件**：CEM 系统「属性导入模版」Excel 文件
**目标位置**：`config/field_spec.yaml`

**替换步骤**：
1. 从 CEM 系统导出最新的「属性导入模版」Excel
2. 使用工具的「导入字段规范」功能自动转换，或手动编辑 `config/field_spec.yaml`
3. 确保 `dict_id` 与数据字典文件中的标识一致

**验证方法**：
```bash
# 检查 YAML 格式是否正确
python -c "import yaml; yaml.safe_load(open('config/field_spec.yaml'))"
```

---

### 2. 数据字典文件（必须）

**源文件**：CEM 系统「租户字典导入模版」Excel 文件
**目标位置**：`dict/data_dict.xlsx`

**格式要求**（v1.2 重大变更）：
- **必须为单 Sheet**
- 第0行：字典类型标识（A1、A2、A3...）
- 第1行：字典类型名称
- 第2行：列头（Code、Details）
- 第3行起：数据行

**示例结构**：
| A | B | C | D | E | F |
|---|---|---|---|---|---|
| A1 | 客户来源 | Code | Details | | |
| | | 01 | 线上 | | |
| | | 02 | 线下 | | |
| A2 | 客户等级 | Code | Details | | |
| | | VIP | VIP客户 | | |
| | | NORMAL | 普通客户 | | |

**验证方法**：
```python
import polars as pl
df = pl.read_excel('dict/data_dict.xlsx')
print(df.head(10))
```

---

### 3. 应用配置文件（可选）

**文件位置**：`config/app_config.yaml`

**可配置项**：
- 去重字段（手机号/邮箱/其他）
- 超时时间
- 日志级别和保留数量

**默认值说明**：
```yaml
# 默认使用手机号作为去重字段
deduplication:
  cross_list:
    - 手机号

# 日志保留最近5份
logging:
  backup_count: 5
```

---

## 部署检查清单

在正式部署前，请执行以下检查：

### 文件存在性检查
- [ ] `config/field_spec.yaml` 存在且格式正确
- [ ] `dict/data_dict.xlsx` 存在且格式正确
- [ ] `config/app_config.yaml` 存在

### 格式验证
- [ ] `field_spec.yaml` 无 YAML 语法错误
- [ ] `data_dict.xlsx` 为单 Sheet 格式
- [ ] 字典标识与 `field_spec.yaml` 中的 `dict_id` 一一对应

### 功能验证
- [ ] 工具启动无报错
- [ ] 字段规范导入功能正常
- [ ] 数据字典加载正常
- [ ] 完整流程可执行

---

## 回滚方案

如需回滚到 mock 文件：
1. 从版本控制（git）恢复正式文件
2. 重启工具
